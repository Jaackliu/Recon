# Personal Finance Dashboard 前端架构与契约设计

本文档定义了 Personal Finance Dashboard 项目的前端页面结构、组件布局以及核心交互与数据展示逻辑。本文件是项目前端架构的**唯一真理来源 (Single Source of Truth)**，后续所有的前端组件划分、状态管理与图表渲染均需严格遵循此规范。

---

## 一、 总体页面结构布局

页面整体采用经典的“左右两栏 + 顶部控制栏”的 Dashboard 布局方案，以提供清晰的层级与全局控制体验：
*   **顶部控制栏 (Top Navigation)**：用于切换全局页面视图（仪表盘或流水明细）。
*   **左侧控制栏 (Left Sidebar)**：用于全局数据维度的筛选（资产选择、时间跨度），悬浮固定。
*   **右侧主内容区 (Main Display Area)**：响应左侧的筛选状态，展示各种核心指标卡片与可视化图表。

---

## 二、 顶部控制栏 (Top Navigation)

控制栏包含两个主要的页面切换 Tab：

### 1. Dashboard (仪表盘视图)
*   **功能**：选中后，右侧主内容区展示常规的图表与面板概览（具体见第四部分）。

### 2. Transactions (交易流水视图)
*   **功能**：选中后，右侧主内容区变为列表视图，列出 `transactions.json` 中的所有交易条目。
*   **展示逻辑**：默认按时间顺序”从近到远”排列。
*   **交互控制**：
    *   **排序控制**：提供下拉菜单或按钮，支持”按时间排序”或”按金额大小排序”。
    *   **过滤控制**：提供筛选器，可勾选单选或多选过滤类型：”仅收入”、”仅支出”、”仅撤销/报销”、”仅内部转账”。
*   **金额正负显示规则**：交易列表中的正负号必须依据 `cashflow_direction` 判定，**与 `type_code` 无关**。
*   **交易列表显示字段**：
    *   **描述**：交易描述信息。
    *   **日期**：交易日期。
    *   **账户别名**：来自 `accounts.json` 的 `alias` 字段。
    *   **交易类型**：使用 `.tag` 样式显示（income/expense/refund/transfer）。
    *   **收支类别**：使用 `.tag.category` 样式显示，与交易类型大小一致。
    *   **金额**：带正负号的交易金额。
    *   **余额**：该笔交易后的帐户余额。
    *   **Hover Tooltip**：鼠标悬停于任意交易行时，行高亮并在光标旁弹出浮窗，展示 `transactions.json` 中该笔交易的全部字段（含 `transaction_id`、`account_code`、`type_code`、`currency`、`raw_text`、`processed_at`、`source_hash` 等）。
    *   *注：此视图依然受左侧控制栏的全局账户和时间筛选影响。*

## 三、 左侧控制栏 (Left Sidebar)

左侧栏为全局筛选器，其状态变更将触发整个 Dashboard 或 Transactions 页面的数据重绘（明确标注不受影响的图表除外）。由于前端不进行复杂数据计算，这里的筛选本质上是对后端提供的全量预计算数据进行前端视角的截取和组装。

### 1. 账户选择器 (Account Selector)
*   **选项**：包含”总资产 (Total Asset)”以及 `data/config/accounts.json` 中定义的所有特定银行账户列表。
*   **计算逻辑契约**：
    *   选择“总资产”时：各项指标（总余额、总收支等）以及所有图表数据，必须是所有单独账户对应项的**直接加和**。
    *   注意：由于跨账户“内部转账”的收入和支出在总额计算时会自动相互抵消，总资产视图下不会凭空产生专门的虚拟账户或虚假交易，只需严格执行加和逻辑即可。

### 2. 货币选择器 (Currency Selector)
*   **位置**：位于账户选择器与时间范围调节器之间。
*   **UI 形式**：与时间范围调节器一致的药丸式按钮组。
*   **选项来源**：
    *   当选择“总资产”时：允许选择“默认”以及所有账户 `supported_currencies` 的并集。
    *   当选择特定账户时：允许选择“默认”以及该账户的 `supported_currencies`。
*   **展示文本**：具体币种选项使用 `data/config/currency.json` 的 `currency_name` 字段展示。
*   **默认币种语义**：
    *   **特定账户 + 默认**：使用该账户在 `data/config/accounts.json` 中的 `default_currency` 视角。
    *   **总资产 + 默认**：使用 `settings.json` 的 `global_default_currency` 视角。
*   **数据绑定契约**：
    *   “默认”使用后端输出的 `default` / `default_local` 数据集。
    *   具体币种使用后端输出的对应 `currency_code` 数据集。

### 3. 时间范围调节器 (Time Range Picker)
*   **特别注意**：时间范围的最后一天为 `transactions.json` 有数据的最后一天，而非当前系统日期。
*   **选项**：支持”1W”（近 7 天）、”1M”（近 30 天）、”3M”（近 90 天）、”6M”（近 180 天）、”1Y”（近 365 天）、”All”（所有时间）以及”Customize”（自定义）。
*   **默认选项**：默认选中”3M”（近 90 天）。
*   **交互**：点击”Customize”时，需弹出日历组件供用户选择起始和结束日期。
*   **联动范围**：随意调节全局时间筛选器后，右侧所有数据和图表必须即刻联动更新（除了 **C. 每日热力图** 和 **D. 月度余额曲线组合图** 明确豁免外）。

### 4. 设置按钮 (Settings Button)

*   **位置**：位于左侧控制栏底部。
*   **UI 形式**：带有齿轮图标的按钮，点击后打开设置弹窗。
*   **设置弹窗内容**：
    *   **主题选择 (Appearance)**：提供三个选项 - “System”（跟随系统）、”Light”（浅色模式）、”Dark”（深色模式）。
    *   **默认货币选择 (Default Currency)**：显示所有可用货币选项，使用 `data/config/currency.json` 的 `currency_symbol` 和 `currency_name` 展示。
*   **数据持久化**：
    *   主题设置保存到 `localStorage` 的 `theme` 键。
    *   默认货币设置保存到 `localStorage` 的 `defaultCurrency` 键。
*   **默认币种语义**：设置的默认货币将用于”总资产 + 默认”视图的货币视角。前端加载 `fx_rate.json` 并读取 UI JSON 中的 `_meta.processor_default_currency`，当用户选择的默认货币与处理器默认货币不同时，自动对 `”default”` 数据集进行运行时汇率转换。

## 四、 右侧主显示区 (Dashboard 视图)

Dashboard 右侧面板按照网页设计草图划分为多个信息模块。以下是各模块的详细展现规范与数据过滤契约。

### A. 余额概览 (Balance Overview)
*   **布局**：左上角首个面板，使用大字体突出当前核心数字。
*   **展示逻辑**：
    *   **当选择“总资产”时**：主标题显示总资产当前总余额。下方列表显示每个银行账户的名称及当前余额。总余额右侧需展示一个**环比变化比例**。
    *   **当选择”特定银行账户”时**：仅显示该所选银行账户的当前余额，余额右侧展示该账户的**环比变化比例**。当货币选择为”默认”时，下方列表显示该账户所有支持货币的余额（汇率转换为账户 `default_currency`），按余额绝对值大小降序排列。数据来源为后端生成的 `ui_currency_breakdown.json`。
*   **环比计算规则**：
    *   若时间选“近 7 天”：用当前 7 天的期末余额对比上一个 7 天的期末余额计算比例。
    *   若时间选“近 30 天”：用当前 30 天期末对比上个 30 天期末计算比例。
    *   若时间选“所有时间”或“自定义日期”：**隐藏/不显示**变化比例。

### B. 现金流概况 (Cash Flow)
*   **布局**：紧邻余额概览面板的右侧。
*   **展示指标**：
    1.  **净流入金额** (Netflow)：流入减流出，可正可负。
    2.  **流入金额** (Inflow)：绝对值为正数。右侧附属显示：`(本期流入金额 / 本期期初余额) * 100%`，带正号。
    3.  **流出金额** (Outflow)：带负号展示。右侧附属显示：`(本期流出金额 / 本期期初余额) * 100%`，带负号。
    4.  **净内部转账** (Net Internal Transfer)：仅限内部转账产生的金额差。若接收转入 > 转出，为正数；反之为负数。
*   **数据契约 (极其重要)**：为**诚实反映现金流**，本模块的计算 **绝对不排除** 任何交易类型（包含撤销、内部转账均需如实计入对应收支）。
*   **方向判定**：所有流入/流出方向均以 `cashflow_direction` 为准，**不参考 `type_code`**。

### C. 每日热力图 (Heatmap)
*   **布局**：顶部行偏右侧模块。
*   **视觉呈现**：类似于 GitHub 贡献图。展示近 3 个月每天的“净流入金额”热力图。每一列代表一周（7 行格子），颜色深浅代表金额大小差异。
*   **数据契约**：为**诚实反映现金流**，本模块的计算 **绝对不排除** 任何交易类型（包含撤销、内部转账均需如实计入对应收支）。
*   **方向判定**：热力图的净流入计算必须基于 `cashflow_direction`。
*   **时间联动契约**：**不受**左侧控制栏的时间范围调节器影响（始终保持近 3 个月）。
*   **交互**：点击任意日期方格，自动将左侧时间范围切换为“自定义”，范围为该日。

### D. 月度余额曲线和收支柱状组合图 (Monthly Combo Chart)
*   **布局**：右上角模块。
*   **视觉呈现**：双 Y 轴组合图。显示近 12 个月的月度数据。
    *   **X 轴**：月份。
    *   **左侧 Y 轴**：对应**折线图**，表示每个月结束时的账户余额。
    *   **右侧 Y 轴**：对应**叠加柱状图**（分正负或分色），表示每月的流入与流出总额。左右 Y 轴尺度自适应，不必相同。
*   **数据契约**：为**诚实反映现金流**，本模块的计算 **绝对不排除** 任何交易类型（包含撤销、内部转账均需如实计入对应收支）。
*   **方向判定**：月度流入/流出必须基于 `cashflow_direction`。
*   **时间联动契约**：**不受**左侧控制栏的时间范围调节器影响（始终保持近 12 个月）。
*   **交互**：点击任意月份的 axisPointer（或同一垂直线上的任意位置），自动将左侧时间范围切换为“自定义”，范围为该月。

### E. 每日余额曲线和收支柱状组合图 (Daily Combo Chart)
*   **布局**：中部横跨面板（通常占据一整行）。
*   **视觉呈现**：双 Y 轴组合图。显示**所选时间周期内每一天**的数据。
    *   **X 轴**：日期。
    *   **左侧 Y 轴**：对应**折线图**，表示每日结束时的账户余额。
    *   **右侧 Y 轴**：对应**叠加柱状图**，表示每日的流入与流出总额。
*   **数据契约**：为**诚实反映现金流**，本模块的计算 **绝对不排除** 任何交易类型（包含撤销、内部转账均需如实计入对应收支）。
*   **方向判定**：每日流入/流出必须基于 `cashflow_direction`。
*   **交互**：点击任意日期的 axisPointer（或同一垂直线上的任意位置），弹出覆盖式详情卡片，展示当天日期、余额、netflow、inflow、outflow，并列出当天全部交易（具体内容、收支类别、交易类型、金额、账户别名、余额），支持按金额排序，并可按交易类型筛选。每条交易行支持 hover 高亮及完整字段浮窗（同 Transactions 视图）。

### F. 资金流向桑基图 (Cash Flow Sankey)
*   **布局**：底部左侧面板。
*   **视觉呈现**：从左向右的四级节点流动关系。
    *   层级 1 (最左)：**收入类别** (来自 transactions.json 的 category)
    *   层级 2 (中左)：**总收入**、*(可能的)* **余额变动**
    *   层级 3 (中右)：**总支出**、*(可能的)* **余额变动**
    *   层级 4 (最右)：**支出类别** (来自 transactions.json 的 category)
*   **动态平衡逻辑契约** (解决收支不平问题)：
    *   **若 支出 > 收入**：在层级 2（中左）新增一个名为”**动用余额**”的节点，它的流量连同”总收入”节点，共同汇聚流向层级 3 的”总支出”节点。
    *   **若 收入 > 支出**：在层级 3（中右）新增一个名为”**结余留存**”的节点，它连同”总支出”节点，共同承接来自层级 2”总收入”节点的流出。
*   **数据契约**：涉及的收支计算分类，**必须排除** `交易类型代码: 3` 和 `交易类型代码: 4`。
*   **交互**：点击层级 1 的收入类别或层级 4 的支出类别节点，弹出覆盖式详情卡片，展示所选时间范围内该类别的全部交易（包含所有类型），支持按金额排序。每条交易行支持 hover 高亮及完整字段浮窗。

### G. 分类占比 (Proportion)

*   **布局**：底部右侧面板（纵向长图块结构）。
*   **视觉呈现**：
    *   **右上角交互**：提供”支出”与”收入”的 Toggle 切换按钮。
    *   **主体部分**：环形/饼状图，根据 `transactions.json` 的 category 展示所选类型（收入或支出）的各类别金额占比。下方附带图例。
*   **数据契约**：占比计算中，**必须排除** `交易类型代码: 3` 和 `交易类型代码: 4`。
*   **交互**：点击环形图扇区，弹出覆盖式详情卡片，展示所选时间范围内该类别的全部交易（包含所有类型），支持按金额排序。每条交易行支持 hover 高亮及完整字段浮窗。
